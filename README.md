# OpenClaw on Google Cloud (Secure GKE Autopilot Template)

> **Disclaimer:** This is not an officially supported Google product.

This repository provides a production-grade, highly secure Terraform template for deploying and hosting the **OpenClaw** AI Agent framework on Google Cloud. 

This architecture is designed around strict **Zero-Trust** principles, enforcing absolute isolation for AI workloads using enterprise-grade Google Cloud security products. It is specifically engineered to run **Gemini 3.1 Flash Preview** via Vertex AI while deeply inspecting and protecting all interactions.

---

## Technical Architecture & Security Posture

This template implements a "defense-in-depth" strategy, ensuring that the OpenClaw agent, its operating environment, and its interactions with Vertex AI are heavily fortified against data exfiltration, prompt injection, and lateral movement.

### 1. Private Networking & Egress Strategy
The entire deployment resides within a fully private Virtual Private Cloud (VPC). 
*   **No Public IPs:** Nodes, pods, and management VMs have no external IP addresses. Private Ingress is strictly enforced via IAP.
*   **Secure Web Proxy (SWP):** All outbound HTTP/HTTPS traffic generated by the OpenClaw AI Agent (including its headless browser tools) is explicitly routed through Google Cloud Secure Web Proxy via the `HTTPS_PROXY` environment variable for deep URL-level transaction auditing.
*   **Cloud Next Generation Firewall (NGFW):** A Global Network Firewall Policy is attached to the VPC. It utilizes **Google Cloud Threat Intelligence (GCTI)** to proactively drop network connections to known malicious IPs, Tor exit nodes, and cryptocurrency miners at the VPC layer, ensuring a robust defense-in-depth architecture alongside SWP.

### 2. Multi-Instance Architecture & Stateful Agents
This template utilizes a dynamic Terraform module approach to support multiple, isolated OpenClaw instances.
*   **Optional Persistence (Stateful vs Ephemeral):** You can choose to deploy agents with long-term memory. By toggling `enable_persistence = true`, the module dynamically provisions a GKE Persistent Volume (PD CSI) to retain conversation history, agent memory, and dynamic configurations across pod restarts. Setting it to `false` creates a completely amnesic, "throwaway" sandbox.
*   **Instance-specific resources:** Each instance generates its own unique Gateway Token (stored in Secret Manager), Internal Load Balancer, IP address, and internal DNS record (e.g., `main.ui.openclaw.internal`).

### 3. AI Security: Model Armor & Sensitive Data Protection (SDP)
Interactions with the Vertex AI Gemini models are actively monitored and filtered at the infrastructure level.
*   **Prompt Injection Detection:** Model Armor evaluates incoming prompts for malicious manipulation or jailbreak attempts.
*   **Data Loss Prevention (DLP):** SDP templates are configured to scan prompts and model responses for PII (e.g., Credit Card numbers).

## Deployment Instructions

### Prerequisites
*   Terraform 1.5+
*   Google Cloud SDK (`gcloud`) configured.

### Step 1: Bootstrap State Storage
1. Navigate to the `terraform/` directory.
2. Apply `00-bootstrap.tf` to create the GCS bucket.
3. Update `backend.tf` with the bucket name.

### Step 2: Initialize & Deploy
To deploy your instances, define them in `terraform/terraform.tfvars`:

```hcl
openclaw_instances = {
  "main" = {
    image              = "us-central1-docker.pkg.dev/your-project/openclaw-repo-prod/openclaw-custom:v1.0.0"
    enable_persistence = true   # Retains memory and conversation history
    storage_size       = "10Gi" # Size of the Persistent Disk
  }
  
  "research-bot" = {
    image              = "us-central1-docker.pkg.dev/your-project/openclaw-repo-prod/openclaw-custom:v1.0.0"
    enable_persistence = false  # Ephemeral sandbox; amnesic after restart
  }
}
```

Then apply the configuration:
```bash
terraform init
terraform plan
terraform apply
```

### Step 3: Accessing the Web Console
The UI for each instance is exposed via an Internal Load Balancer with a unique DNS record.

1. **Obtain Tokens:**
   Gateway tokens are automatically generated, stored in Google Secret Manager, and securely injected into the pods via the CSI driver. To retrieve the tokens for login:
   ```bash
   terraform output -json gateway_tokens
   ```
2. **Access via IAP Tunnel:**
   Establish a secure tunnel to your desired instance (e.g., the `main` instance):
   ```bash
   gcloud compute ssh openclaw-bastion-prod \
       --tunnel-through-iap \
       --zone us-central1-a \
       --project <your-project-id> \
       -- -N -L 18789:main.ui.openclaw.internal:18789 
   ```
   *(To access `research-bot`, change the tunnel target to `research-bot.ui.openclaw.internal`)*
   
   Navigate to `http://localhost:18789` in your browser and enter the corresponding token.

## Skill & Configuration Management (GitOps)
*   **Custom Docker Images:** Skills and OS dependencies (like Playwright, Xvfb, and Chromium for headless web scraping) are pre-installed in a custom Docker image located in `docker-build/`.
*   **Configuration:** The `openclaw.json` is generated via Terraform templates (`terraform/templates/openclaw.json.tpl`) and injected into the container dynamically.
*   **Security:** Traffic to the OpenClaw service is proxied via a sidecar `socat` container to securely handle local-only bindings.
