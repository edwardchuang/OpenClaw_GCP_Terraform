# OpenClaw on Google Cloud (Secure GKE Autopilot Template)

> **Disclaimer:** This is not an officially supported Google product.

This repository provides a production-grade, highly secure Terraform template for deploying and hosting the **OpenClaw** AI Agent framework on Google Cloud. 

This architecture is designed around strict **Zero-Trust** principles, enforcing absolute isolation for AI workloads using enterprise-grade Google Cloud security products. It is specifically engineered to run **Gemini 3.1 Flash Preview** via Vertex AI while deeply inspecting and protecting all interactions.

---

## Technical Architecture & Security Posture

This template implements a "defense-in-depth" strategy, ensuring that the OpenClaw agent, its operating environment, and its interactions with Vertex AI are heavily fortified against data exfiltration, prompt injection, and lateral movement.

### 1. Private Networking & Egress Strategy
The entire deployment resides within a fully private Virtual Private Cloud (VPC). 
*   **No Public IPs:** Nodes, pods, and management VMs have no external IP addresses. Private Ingress is strictly enforced.
*   **Cloud NAT:** A standard Cloud NAT is provided for underlying OS/node-level egress.
*   **Secure Web Proxy (SWP):** All outbound HTTP/HTTPS traffic generated by the OpenClaw AI Agent is explicitly routed through Google Cloud Secure Web Proxy via the `HTTPS_PROXY` environment variable for deep URL-level transaction auditing.
*   **Cloud Next Generation Firewall (NGFW):** A Global Network Firewall Policy is attached to the VPC. It utilizes **Google Cloud Threat Intelligence (GCTI)** to proactively drop network connections to known malicious IPs, Tor exit nodes, and cryptocurrency miners at the VPC layer, ensuring a robust defense-in-depth architecture alongside SWP.

### 2. Multi-Instance Architecture
This template utilizes a Terraform module approach to support multiple, isolated OpenClaw instances.
*   **Instance-specific resources:** Each instance has its own Deployment, Internal Load Balancer, IP address, and API Token.
*   **Shared resources:** The GKE cluster, VPC, and Cloud logging infrastructure are centrally managed.

### 3. AI Security: Model Armor & Sensitive Data Protection (SDP)
Interactions with the Vertex AI Gemini models are actively monitored and filtered at the infrastructure level.
*   **Prompt Injection Detection:** Model Armor evaluates incoming prompts for malicious manipulation or jailbreak attempts.
*   **Data Loss Prevention (DLP):** SDP templates are configured to scan prompts and model responses for PII.

## Deployment Instructions

### Prerequisites
*   Terraform 1.5+
*   Google Cloud SDK (`gcloud`) configured.

### Step 1: Bootstrap State Storage
1. Navigate to the `terraform/` directory.
2. Apply `00-bootstrap.tf` to create the GCS bucket.
3. Update `backend.tf` with the bucket name.

### Step 2: Initialize & Deploy
To deploy your instances, define them in `terraform/terraform.tfvars`:

```hcl
openclaw_instances = {
  "main" = {
    image         = "us-central1-docker.pkg.dev/your-project/openclaw-repo-prod/openclaw-custom:v1.0.0"
    gateway_token = "unused" # Secret managed via GSM
  }
  # Add more instances here
  "secondary" = {
    image         = "us-central1-docker.pkg.dev/your-project/openclaw-repo-prod/openclaw-custom:v1.0.0"
    gateway_token = "unused"
  }
}
```

Then apply the configuration:
```bash
terraform init
terraform plan
terraform apply
```

### Step 3: Secret Management
The application secrets (e.g., `TELEGRAM_BOT_TOKEN`, `GATEWAY_TOKEN`) are managed via **Google Secret Manager (GSM)**.
1. Create the secret in GCP:
   ```bash
   gcloud secrets create openclaw-telegram-token-prod --replication-policy="automatic" --project="your-project"
   echo -n "YOUR_TOKEN_HERE" | gcloud secrets versions add openclaw-telegram-token-prod --data-file=- --project="your-project"
   ```
2. Terraform will automatically sync these to your GKE Pods using the **CSI Driver**.

### Step 4: Accessing the Web Console
The UI is exposed via an Internal Load Balancer.

1. **Obtain Tokens:**
   ```bash
   terraform output -raw gateway_token
   ```
2. **Access via IAP Tunnel:**
   ```bash
   gcloud compute ssh openclaw-bastion-prod \
       --tunnel-through-iap \
       --zone us-central1-a \
       --project <your-project-id> \
       -- -N -L 18789:main.ui.openclaw.internal:18789 
   ```
   Navigate to `http://localhost:18789` in your browser.

## Skill & Configuration Management (GitOps)
*   **Custom Docker Images:** Skills (e.g., summary, workspace tools) are pre-installed in a custom Docker image located in `docker-build/`.
*   **Configuration:** The `openclaw.json` is generated via Terraform templates in `terraform/templates/openclaw.json.tpl`.
*   **Security:** Traffic to the OpenClaw service is proxied via a sidecar `socat` container to securely handle local-only bindings.
